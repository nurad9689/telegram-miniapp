# Миграция базы данных для поддержки Telegram авторизации

## Изменения в схеме базы данных

### Таблица `participants`
Добавлено новое поле:
- `telegram_id` (INTEGER, UNIQUE, NOT NULL) - ID пользователя в Telegram

## Как выполнить миграцию

### Вариант 1: Удалить и пересоздать базу данных (для разработки)

```bash
# Удалите файл базы данных (обычно sports_events.db или подобный)
rm sports_events.db

# Перезапустите сервер - таблицы будут созданы автоматически с новой схемой
python main.py
```

### Вариант 2: Ручная миграция существующей базы данных

```sql
-- Добавить колонку telegram_id
ALTER TABLE participants ADD COLUMN telegram_id INTEGER;

-- Обновить существующие записи (если нужно)
-- Для существующих участников можно установить временные значения
UPDATE participants SET telegram_id = 0 WHERE telegram_id IS NULL;

-- Сделать колонку NOT NULL
-- Сначала убедитесь, что все записи имеют значение
-- Затем выполните (для SQLite это сложнее, может потребоваться пересоздание таблицы)

-- Создать уникальный индекс
CREATE UNIQUE INDEX ix_participants_telegram_id ON participants(telegram_id);
```

### Вариант 3: Использование Alembic (рекомендуется для продакшена)

Если вы используете Alembic для миграций:

```bash
# Создать новую миграцию
alembic revision --autogenerate -m "Add telegram_id to participants"

# Применить миграцию
alembic upgrade head
```

## Проверка после миграции

После миграции убедитесь, что:

1. Таблица `participants` имеет колонку `telegram_id`
2. Колонка `telegram_id` имеет уникальный индекс
3. Приложение успешно создаёт новых пользователей через Telegram

## Тестирование

1. Запустите бэкенд: `python main.py`
2. Запустите фронтенд: `npm start`
3. Откройте приложение в Telegram
4. Новый пользователь должен быть автоматически создан в базе данных
5. При создании события должен использоваться реальный `creator_id` пользователя
